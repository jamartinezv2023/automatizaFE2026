# Estrategia E2E

## Objetivo
Validar flujos críticos del sistema LogiTrack usando Serenity BDD, Cucumber y Screenplay.

## Flujos estabilizados
- Registro de cliente
- Cierre de sesión desde dashboard

## Flujo con hallazgo
- Login directo: documentado en docs/hallazgos/login-no-redirecciona.md

## Criterios
- Pruebas independientes
- Datos dinámicos
- Evidencia en reportes Serenity
- Selectores basados en DOM real
- Esperas explícitas
